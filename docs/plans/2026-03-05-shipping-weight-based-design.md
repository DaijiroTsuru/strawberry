# 重量ベース送料計算 + カート梱包数表示 設計

## 背景

津留いちご園では送料固定（九州1,300円/九州外2,300円）で運用しているが、1梱包に入る重量には上限がある。各バリアントが1梱包の上限であり、複数商品をカートに入れると複数梱包＝送料が倍数になる必要がある。

### 制約
- Shopify Basicプラン（Carrier Service API使用不可）
- 各バリアント = 1梱包（バリアントが重量上限）
- いちごとお米は同梱不可（別梱包）
- カート内の商品数（数量含む）= 梱包数 = 送料の倍数
- ユーザーは複数商品をカートに入れてOK

## 設計

### 1. Shopify管理画面の設定（手動）

#### 1-1. 全バリアントの重量を1kgに統一
Shopify管理画面で全商品バリアントの重量を1kgに設定する。これにより商品数がそのまま総重量（kg）になり、重量ティアと1:1で対応する。

#### 1-2. 配送ゾーンと重量ティアの設定

**ゾーン:**
| ゾーン | 対象県 |
|--------|--------|
| 九州 | 福岡・佐賀・長崎・熊本・大分・宮崎・鹿児島 |
| その他 | 上記以外の都道府県 |

**重量ティア（各ゾーン）:**
| 重量範囲 | 九州 | その他 |
|----------|------|--------|
| 0〜1kg | ¥1,300 | ¥2,300 |
| 1.01〜2kg | ¥2,600 | ¥4,600 |
| 1.01〜3kg | ¥3,900 | ¥6,900 |
| ... (1kgごと最大10段階) | +¥1,300 | +¥2,300 |

### 2. CartDrawer.tsx のコード変更

フッターの合計金額の下に、梱包数の案内を追加する。

**表示条件:** 梱包数が2以上の場合のみ表示（1梱包は当たり前なので不要）

**梱包数の計算:**
```
梱包数 = cartItems.reduce((sum, { node }) => sum + node.quantity, 0)
```

**表示イメージ:**
```
📦 N梱包での配送になります
※送料は梱包数分かかります。正確な送料はチェックアウト時に確定します。
```

### 変更しないもの
- `farmInfo.ts` — 送料金額はShopify管理画面が正とし、フロントにハードコードしない
- カートロジック — 複数商品の追加制限はしない
- その他ページコンポーネント — 変更なし

## ユーザー体験フロー

1. 複数商品をカートに追加
2. カートに「N梱包での配送になります」と表示
3. チェックアウトで住所入力 → Shopifyが重量ティアから正確な送料を計算・表示
4. ユーザーが送料を確認して決済
